# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately. Do not
open a public GitHub issue for security problems.

Email: smara.putra2001@gmail.com

Please include:

- A description of the vulnerability and its impact
- Steps to reproduce (proof of concept if possible)
- Affected versions or components

You can expect an acknowledgement within a few days. Please allow reasonable
time for a fix before any public disclosure.

## Scope

This is an educational and portfolio project, not a hosted service with a
formal SLA. Reports are addressed on a best-effort basis. The most useful
reports concern authentication, authorization, data handling, dependency
vulnerabilities, and secret exposure.

## Out of Scope

- Issues in third-party dependencies that already have a published advisory
  (use Dependabot / `npm audit` instead)
- Findings that require physical access to a user's device
- Reports against the public Binance, Upstash, or other upstream APIs
